Birthday or Death Day?

Yello, Hello! Today, I'm going to be working on Project 3: Art Meets Engineering. In this project, I was assigned to make something that combines artistry with the Arduino. For example, one could make an etch-a-sketch using a potentiometer and the Processing software as a canvas. However, I decided to go more experimental with it. When thinking up an idea for this project, I wanted to make something that had an underlying meaning to it. Like there's something more that meets the eye. So I came up with Birthday. I have this running "joke" whenever it's someone's birthday where I congratulate them on being one year closer to their death. And so sparked my idea for Project 3, a morbid, dark humor-esque game where you have to blow on a candle, increasing in age, but there is a timer that continuously ticks down. If the timer goes to 0, you're dead.

The parts I use are very simple: an LED to detect whenever a candle has been "blown", 3 jumper wires connected to power, the anode of the LED, the cathode of the LED to digital Pin 5, 220 ohm resistor, and tin foil and paper to make a paper candle and extend the contact range with the tin foil.

It didn't feel like a birthday party so I added balloons and candles to get that happy feeling. Now that we have those components in place, having the "Death" timer surrounded by a bunch balloons and celebratory decorations, the dichotomous feelings are now established.

Arduino Code

The process was very similar to Project 2. I really enjoyed being able to serial communicate with other programs, so I wanted to do that again. As a GDD major, gamifying concepts is just what I like to do.

The code for the Arduino is very short. The LED that has a cable connected to the cathode which connects to digital Pin 5 on the UNO. The way this works is that there is a 1 cable connected to power and another connected the anode of the LED. When the 2 cables connect, the LED lights up and the resistor that connects to ground makes a viable circuit. The 3rd cable connects to the LED's cathode, which allows me to be able to detect when the connection has been made and from there I can use the data to my advantage.

blowState reads in the data collected from the LED and if the LED has been turned on and the previousBlowState was off, then it will print out the word BLOW to the serial console once. I want it to be once because for my project, the age of the player goes up by one. If it wasn't there, then it would print multiple BLOWs to the console, therefore, increasing the age by a lot more than 1. Even then, 2 to 3 BLOWs still printed, so the delay(100) guarantees only 1 BLOW.

Unity Code

This code is also similar to Project 2. There are 3 main variables to keep track of:

  1. age - The age of the player.
  2. deathTimer - The timer that leads to the player's inevitable death.
  3. isTimerRunning - Boolean to track timer

The first chunk of code just detects if it reads in the string "BLOW", and if it does, increase the age of the player and deathTimer by 1. Why increase the deathTimer you might ask? Well, to make the game more playable obviously! It's not necessarily fun if no matter what you do, you die (even though that is the case IRL). It makes it more challenging and players will want to see how far they can go without dying.

The next chunk is keeping track of the timer and updating the text on screen. While the deathTimer is greater than 0, just keep decrementing it by the time, if it does cross over to the negatives, then just return false on the isTimerRunning boolean (which stops the timer), and keep the timer at 0. When the isTimerRunning is false, change the text to the "game over" text.
