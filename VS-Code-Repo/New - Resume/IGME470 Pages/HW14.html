So let's get right into it!

Morty, Look At Me! I'm Changing Colors!

The circuitry was painfully easy--it only requires a potentiometer and a couple of jumper cables to connect to power, ground, and AO as the input. The programming part was the meat of this project. There's really nothing more I can say on the building of the circuit, so let's go to the code!

Code

Let's start with the Arduino code. In order to start serial communication, you need to write Serial.begin() in start. With this, you can write Serial.println() if you wanted to debug some things (which is how I used it), but in this case, we are going to use another function of Serial, which is Serial.write(). This allows to us to send information over the serial connection, in a value range of 0-255, but analog pins can read and send values from 0 - 1023 so we have to scale it down by dividing by 4. After sending the information, we use a delay of 1/10 of a second as to not overload the system and cause lag.

Now in Processing world, Processing doesn't automatically have serial communication capabilities so we have to give it that by importing processing.serial. From there, we have to create a Serial object that will hold the ncessessary information to process the serial connection, such as which application it will be speaking to (Processing), which serial port it will communicate over (COM3), and at what speed (9600 bauds). We load in an image on the internet, and if there is any values coming from the port the serial connection is on, then we read it out and store it in bgColor which then would be applied to the hue of the background, shown on line 44.
